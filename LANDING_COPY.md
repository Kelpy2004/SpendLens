## Body Copy

Your AI stack probably grew one team at a time. Cursor here, Copilot there, Claude for product, ChatGPT for operations, Gemini for research, and API usage sitting somewhere else entirely. None of that is reckless. It is just what happens when useful tools spread faster than procurement.

SpendLens gives you a clean audit before renewal season. Enter the tools, tiers, monthly spend, and seats. We compare the stack against current pricing and hardcoded finance rules, then show the savings we can defend.

If the audit finds less than $100/month, we say you are spending well. If it finds more than $500/month, we show why Credex should centralize the cleanup.

## FAQs

### Is this an AI-generated recommendation?

No. The savings logic is hardcoded TypeScript using current vendor pricing and explicit rules. AI is only used to write a short summary paragraph after the math is done, and there is a deterministic fallback if the API fails.

### Why do you ask for email after the audit?

Because the audit should earn trust before asking for anything. Email capture only appears on the result page so users can save the audit and get a confirmation email.

### Do public audit links include my company or email?

No. Public audit URLs only include spend inputs: tools, tiers, seats, spend, team size, and use case. Email, company, and role are submitted separately to the backend and never encoded into the share URL.

### What counts as real savings?

Real savings must be explainable to finance. Same-vendor downgrades use list-price math. Alternative recommendations need meaningful savings, not a $4/month difference. API spend is marked for usage review unless there is enough token-level data to defend the claim.

### Who should use SpendLens?

The best fit is a 10 to 80 person SaaS company using multiple AI tools across engineering, product, and operations. If your team has one ChatGPT Plus account and no API spend, the audit will be honest but probably not dramatic.
